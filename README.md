live site: https://file-explorer-two-flax.vercel.app/

what is done here:
1. Represent the file system in a tree-like hierarchy.
2. A folder can contain both subfolders and files.
3. Each folder/file should have a unique identifier, name, and type.
4. Store the hierarchy in the database so it persists across reloads.
5. Create, delete, navigate, edit done
6. Text file open and edit
7. Img preview
8. Responsiveness

#doubleclick on the file, folder, img to open it.
